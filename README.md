# Тестовое задание для соискателя на вакансию «Инженер-программист»

## 1. Bracket Checker

**Задача:**  
Реализовать алгоритм определения правильной скобочной последовательности в выражении.

**Примеры:**  
- `{(x * y) + (2 * (x + y))} * [y + 3]` — правильная скобочная последовательность.  
- `({((x * y) + [2 * (x - y)]})` — неправильная скобочная последовательность.  
- `(x * y) + ) 2 * [x - y](` — неправильная скобочная последовательность.  

**Формат входных данных:**  
Текстовый файл, содержащий несколько выражений (по одному в каждой строке). Подан программе через аргументы: "src/input_data/Brackets.txt".  

**Формат вывода:**  
Для каждого выражения вывести:  
```
<выражение> - правильная скобочная последовательность
```
или  
```
<выражение> - неправильная скобочная последовательность
```

**Подход к решению:**  
- Метод `checkBrackets(String line)` использует стек для отслеживания открывающих скобок.  
- Проходит по каждому символу строки:  
  - Если символ — открывающая скобка, помещает её в стек.  
  - Если символ — закрывающая скобка, проверяет соответствие с последней открывающей скобкой (через `isMatchingPair`).  
- Возвращает `true`, если стек пуст (все скобки закрыты), иначе `false`.

## 2. Sadle Point Checker

**Задача:**  
Написать программу, выполняющую поиск седловой точки в заданной матрице.  

**Определение:**  
Седловая точка матрицы — это элемент, который одновременно:  
- является минимальным в своей строке,  
- является максимальным в своём столбце.  

**Подход к решению:**  
Осуществляется проход по элементам матрицы и проверка условий для признания элемента седловой точкой.

## 3. Snake/Camel Case Converter

**Задача:**  
Дана строка-идентификатор переменной, содержащая только латинские буквы и символы подчеркивания `_`.  
- **snake_case**: начинается и заканчивается строчной буквой, содержит только строчные буквы и одиночные `_`.  
- **camelCase**: начинается со строчной буквы, содержит строчные и прописные буквы.  

Программа должна преобразовывать строку из одного формата в другой. Если строка не соответствует ни одному формату, выводится ошибка.  

**Примеры:**  
- `"some_variable"` → `"someVariable"`  
- `"tryToConvertMe"` → `"try_to_convert_me"`  
- `"unchanged"` → `"unchanged"`  
- `"InvalidMethod"` → `"Error!"`  
- `"bad_VarName"` → `"Error!"`  

**Подход к решению:**  
- **`checkSnakeCase`**:  
  - Проверяет, что строка начинается и заканчивается строчной буквой.  
  - Проверяет, что все символы — строчные буквы или одиночные `_`.  
  - Проверяет отсутствие двойных `_`.  
- **`checkCamelCase`**:  
  - Проверяет, что строка начинается со строчной буквы.  
  - Проверяет, что остальные символы — буквы (строчные или прописные).  
- **`snakeToCamel`**:  
  - Удаляет `_` и делает следующую букву заглавной.  
- **`camelToSnake`**:  
  - Вставляет `_` перед заглавными буквами и приводит их к строчным.

## 4. Box Builder

**Задача:**  
Даны 6 прямоугольных фанерных листов. Определить, можно ли из них собрать коробку (прямоугольный параллелепипед).  

**Формат входных данных:**  
Текстовый файл с 6 строками, где каждая строка содержит 2 целых числа (длина и ширина листа, от 1 до 1000), разделённых пробелом.  

**Формат вывода:**  
`"Возможно"` или `"Невозможно"`.  

**Подход к решению:**  
Проверка осуществляется последовательно:  
1. Если уникальных размеров листов больше 3 → `"Невозможно"`.  
2. Если уникальный размер только один → возможно, если все листы — квадраты.  
3. Если уникальных размеров два → возможно при условии:  
   - 2 листа в одной группе и 4 в другой,  
   - листы меньшей группы — квадраты со стороной, равной одной из сторон листов большей группы.  
4. Если уникальных размеров три → возможно при равномерном распределении листов и возможности образовать 3 смежные грани параллелепипеда.
